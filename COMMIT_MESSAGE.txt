v0.2.0 - Sprint 3.1 植物管理功能

新增功能：
- JSON 文件数据持久化
- 添加植物：回复"添加 植物名称"
- 查看植物列表：回复"1"或"我的植物"
- 删除植物：回复"删除 植物名称"
- 用户数据自动初始化和管理

技术改进：
- 使用 fs.promises 异步文件操作
- 数据存储在 data/users.json
- 每个用户独立的植物数据
- 自动创建数据目录

使用示例：
- 添加 绿萝
- 添加 多肉植物
- 1（查看列表）
- 删除 绿萝

- VERSION.md: 添加版本记录
- data/.gitkeep: 创建数据目录
